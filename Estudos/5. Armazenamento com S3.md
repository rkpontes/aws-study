# Etapa 5 — Armazenamento com S3

Nesta etapa, vamos tirar os arquivos da EC2 e armazená-los no **Amazon S3**.

Isso complementa o que já foi construído antes:

* na [Etapa 3](./3.%20Publicando%20uma%20aplicação.md), a aplicação ficou publicada na EC2 com **frontend em Nginx** e **backend Node.js/Express**;
* na [Etapa 4](./4.%20Banco%20de%20dados%20com%20Amazon%20RDS.md), o PostgreSQL foi movido para o **Amazon RDS**.

Agora a arquitetura passa a ser:

```text
Internet
   ↓
EC2
├── Frontend (Nginx)
└── Backend (Node.js/Express)
        ↓
Amazon S3

RDS PostgreSQL
   ↓
Guarda apenas a chave do arquivo
```

O **S3 não fica dentro da EC2**. Ele é um serviço separado, pensado para armazenar arquivos com durabilidade alta e sem depender do disco da instância.

---

## 1. Entendendo o problema atual

Se hoje a aplicação salvar imagens diretamente no disco da EC2, você terá alguns riscos:

* trocar a instância pode fazer os arquivos sumirem;
* escalar para mais de uma EC2 vira um problema, porque cada máquina teria seu próprio disco;
* backup de arquivos fica manual;
* aplicação e armazenamento continuam acoplados.

Com S3, o backend continua recebendo o upload, mas o arquivo deixa de ficar preso à máquina.

```text
Usuário
   ↓
Frontend
   ↓
Backend na EC2
   ↓
S3

RDS
   ↓
salva somente a referência do arquivo
```

---

## 2. Criar o bucket

No Console da AWS:

```text
S3 → Create bucket
```

Use estas configurações:

1. Escolha um nome globalmente único, por exemplo:

```text
app-imagens-dev
```

2. Selecione a mesma região da EC2 e do RDS.
3. Mantenha **Block all public access** ativado.
4. Ative o **Versioning**.
5. Crie o bucket.

O versionamento ajuda a recuperar versões antigas de um objeto caso ele seja sobrescrito ou removido por engano. ([Documentação AWS][1])

---

## 3. Permitir que a EC2 acesse o S3

O jeito mais prático e seguro de fazer isso é usar uma **IAM Role associada à EC2**.

Na prática, essa role funciona como uma "identidade" da sua instância. Quando o backend rodar dentro da EC2, ele poderá enviar e ler arquivos do S3 sem precisar salvar chave de acesso no código ou no `.env`.

### Passo 1. Criar a policy de acesso ao bucket

No Console da AWS:

```text
IAM → Policies → Create policy
```

Escolha a aba **JSON** e cole uma policy com acesso somente ao seu bucket:

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": [
        "s3:PutObject",
        "s3:GetObject",
        "s3:DeleteObject"
      ],
      "Resource": "arn:aws:s3:::app-imagens-dev/*"
    }
  ]
}
```

Troque `app-imagens-dev` pelo nome real do bucket que você criou.

Depois:

1. Clique em **Next**.
2. Dê um nome para a policy, por exemplo:

```text
ec2-s3-app-policy
```

3. Clique em **Create policy**.

Se em algum momento sua aplicação precisar listar arquivos do bucket, adicione também a permissão `s3:ListBucket`:

```json
{
  "Effect": "Allow",
  "Action": "s3:ListBucket",
  "Resource": "arn:aws:s3:::app-imagens-dev"
}
```

### Passo 2. Criar a role para a EC2

Agora crie a role que será usada pela instância:

```text
IAM → Roles → Create role
```

Configure assim:

1. Em **Trusted entity type**, escolha **AWS service**.
2. Em **Use case**, escolha **EC2**.
3. Clique em **Next**.
4. Procure a policy criada no passo anterior. Se você não trocou deve ser `app-imagens-dev`.
5. Marque essa policy.
6. Clique em **Next**.
7. Dê um nome para a role, por exemplo:

```text
ec2-s3-app-role
```

8. Clique em **Create role**.

### Passo 3. Associar a role à sua instância

Com a role pronta, falta ligar essa permissão à EC2 onde o backend está rodando.

No Console da AWS:

```text
EC2 → Instances → selecione sua instância
```

Depois siga:

1. Clique em **Actions**.
2. Vá em **Security**.
3. Clique em **Modify IAM role**.
4. Selecione a role criada:

```text
ec2-s3-etapa-3-app-role
```

5. Clique em **Update IAM role**.

### Passo 4. Entender o que acontece depois

Depois que a role estiver associada, o SDK da AWS dentro do Node.js passa a buscar as credenciais automaticamente na própria EC2.

Isso significa que você **não precisa** configurar estas variáveis na aplicação:

```env
AWS_ACCESS_KEY_ID=
AWS_SECRET_ACCESS_KEY=
```

Você só precisa informar dados como região e nome do bucket.

## 6. Criar uma tabela para guardar a referência do arquivo

Na etapa 4 já existe a tabela `app_messages`, criada pelo dump em:

[`Projetos/etapa-4/dump-etapa-4.sql`](/Users/raphaelpontes/Documents/Estudos/AWS/Projetos/etapa-4/dump-etapa-4.sql)

Conecte ao banco 

```bash
psql \
  --host=app-postgres.abc123.us-east-1.rds.amazonaws.com \
  --port=5432 \
  --username=etapa_3_app_user \
  --dbname=etapa_3_app
```

Digite a senha quando solicitado.

Agora podemos criar uma tabela separada para uploads:

```sql
CREATE TABLE IF NOT EXISTS public.app_images (
    id BIGSERIAL PRIMARY KEY,
    original_name VARCHAR(255) NOT NULL,
    storage_key VARCHAR(500) NOT NULL UNIQUE,
    content_type VARCHAR(120) NOT NULL,
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
```

Observe que estamos salvando a **chave do objeto** e não a URL completa.

Exemplo:

```text
uploads/1721400000000-foto.png
```

Isso facilita trocar bucket, domínio ou colocar CloudFront depois sem regravar dados no banco.

## 7. Exemplo de implementação no `server.js`

Um exemplo mínimo, mas já mais seguro para adaptar o backend Express:

```js
const express = require("express");
const multer = require("multer");
const { randomUUID } = require("node:crypto");
const { Pool } = require("pg");
const { S3Client, PutObjectCommand, GetObjectCommand } = require("@aws-sdk/client-s3");
const { getSignedUrl } = require("@aws-sdk/s3-request-presigner");

const app = express();
const port = 3000;

if (!process.env.AWS_REGION || !process.env.AWS_S3_BUCKET) {
  throw new Error("Defina AWS_REGION e AWS_S3_BUCKET antes de iniciar o backend.");
}

const upload = multer({ storage: multer.memoryStorage() });

const s3Client = new S3Client({
  region: process.env.AWS_REGION
});

const pool = new Pool({
  host: process.env.DB_HOST,
  port: Number(process.env.DB_PORT || 5432),
  database: process.env.DB_NAME,
  user: process.env.DB_USER,
  password: process.env.DB_PASSWORD,
  ssl: {
    rejectUnauthorized: false
  }
});

app.get("/health", async (request, response) => {
  response.json({
    status: "ok",
    service: "backend"
  });
});

app.get("/api/message", async (request, response) => {
    try {
        const result = await pool.query(`
            SELECT title, content, created_at
            FROM app_messages
            WHERE slug = 'mensagem-principal'
            LIMIT 1
        `);

        if (result.rowCount === 0) {
            return response.status(404).json({
                error: "Mensagem não encontrada"
            });
        }

        const message = result.rows[0];

        response.json({
            message: message.content,
            title: message.title,
            createdAt: message.created_at
        });
    } catch (error) {
        response.status(500).json({
            error: "Erro ao consultar o banco de dados"
        });
    }
});

app.post("/api/images", upload.single("image"), async (request, response) => {
  try {
    if (!request.file) {
      return response.status(400).json({
        error: "Arquivo nao enviado."
      });
    }

    const key = `uploads/${randomUUID()}-${request.file.originalname}`;

    await s3Client.send(
      new PutObjectCommand({
        Bucket: process.env.AWS_S3_BUCKET,
        Key: key,
        Body: request.file.buffer,
        ContentType: request.file.mimetype
      })
    );

    const result = await pool.query(
      `
        INSERT INTO public.app_images (original_name, storage_key, content_type)
        VALUES ($1, $2, $3)
        RETURNING id, original_name, storage_key, content_type, created_at
      `,
      [request.file.originalname, key, request.file.mimetype]
    );

    return response.status(201).json(result.rows[0]);
  } catch (error) {
    console.error("Erro ao enviar imagem para o S3:", error);

    return response.status(500).json({
      error: "Falha ao enviar imagem."
    });
  }
});

app.get("/api/images/:id/url", async (request, response) => {
  try {
    const result = await pool.query(
      `
        SELECT id, storage_key
        FROM public.app_images
        WHERE id = $1
      `,
      [request.params.id]
    );

    if (result.rowCount === 0) {
      return response.status(404).json({
        error: "Imagem nao encontrada."
      });
    }

    const image = result.rows[0];

    const url = await getSignedUrl(
      s3Client,
      new GetObjectCommand({
        Bucket: process.env.AWS_S3_BUCKET,
        Key: image.storage_key
      }),
      { expiresIn: 300 }
    );

    return response.json({
      id: image.id,
      url
    });
  } catch (error) {
    console.error("Erro ao gerar URL da imagem:", error);

    return response.status(500).json({
      error: "Falha ao gerar URL da imagem."
    });
  }
});

app.listen(port, "0.0.0.0", () => {
  console.log(`Backend executando na porta ${port}`);
});
```

```html
<!DOCTYPE html>

        }

        h1 {
            margin-top: 0;
        }

        form {
            margin-top: 24px;
            display: grid;
            gap: 12px;
        }

        button {
            width: fit-content;

            font-weight: bold;
        }

        pre {
            padding: 16px;
            overflow: auto;

            border-radius: 8px;
        }

        img {
            display: block;
            max-width: 100%;
            margin-top: 16px;
            border-radius: 8px;
        }
    </style>
</head>

<body>
    <main>
        <h1>Aplicação publicada na AWS</h1>

        <p>O frontend está sendo servido pelo Nginx dentro da EC2.</p>

        <button id="load-message">
            Consultar backend
        </button>

        <pre id="result">Clique no botão para chamar a API.</pre>

        <form id="upload-form">
            <label for="image">Escolha uma imagem para enviar ao S3:</label>
            <input id="image" name="image" type="file" accept="image/*" required>

            <button type="submit">
                Enviar imagem
            </button>
        </form>

        <img id="preview" alt="Preview da imagem enviada" hidden>
    </main>

    <script>
        const button = document.getElementById("load-message");
        const form = document.getElementById("upload-form");
        const result = document.getElementById("result");
        const preview = document.getElementById("preview");

        button.addEventListener("click", async () => {
            result.textContent = "Carregando...";

            try {
                const response = await fetch("/api/message");

                if (!response.ok) {
                    throw new Error(`HTTP ${response.status}`);
                }

                const data = await response.json();
                result.textContent = JSON.stringify(data, null, 2);
            } catch (error) {
                result.textContent = `Erro: ${error.message}`;
            }
        });

        form.addEventListener("submit", async (event) => {
            event.preventDefault();

            const fileInput = document.getElementById("image");
            const file = fileInput.files[0];

            if (!file) {
                result.textContent = "Selecione um arquivo antes de enviar.";
                return;
            }

            const formData = new FormData();
            formData.append("image", file);

            result.textContent = "Enviando imagem...";
            preview.hidden = true;

            try {
                const uploadResponse = await fetch("/api/images", {
                    method: "POST",
                    body: formData
                });

                if (!uploadResponse.ok) {
                    throw new Error(`HTTP ${uploadResponse.status}`);
                }

                const uploadData = await uploadResponse.json();

                const urlResponse = await fetch(`/api/images/${uploadData.id}/url`);

                if (!urlResponse.ok) {
                    throw new Error(`HTTP ${urlResponse.status}`);
                }

                const urlData = await urlResponse.json();

                result.textContent = JSON.stringify(
                    {
                        upload: uploadData,
                        temporaryUrl: urlData.url
                    },
                    null,
                    2
                );

                preview.src = urlData.url;
                preview.hidden = false;
            } catch (error) {
                result.textContent = `Erro: ${error.message}`;
            }
        });
    </script>
</body>

</html>
```

Nesse frontend, o fluxo fica assim:

1. o usuário escolhe um arquivo no navegador;
2. a página envia o `multipart/form-data` para `/api/images`;
3. o backend grava o arquivo no S3 e salva os metadados no RDS;
4. depois a página chama `/api/images/:id/url`;
5. por fim, a imagem é exibida usando a URL temporária retornada pelo backend.

Repare em dois pontos importantes desse exemplo:

1. O backend falha ao iniciar se `AWS_REGION` ou `AWS_S3_BUCKET` não estiverem definidos.
2. As rotas usam `try/catch` para evitar que um erro do S3 derrube a resposta sem contexto.

Esse exemplo mantém o mesmo modelo das etapas anteriores:

* o frontend continua chamando o backend por `/api`;
* o backend continua centralizando a lógica;
* o banco continua guardando apenas metadados;
* o S3 passa a ser o armazenamento dos arquivos.

## 8. Como testar

Depois de subir a nova versão da aplicação na EC2:

1. envie um arquivo para `POST /api/images`;
2. confirme no console do S3 que o objeto apareceu no bucket;
3. confirme no PostgreSQL que a tabela `app_images` recebeu um registro;
4. chame `GET /api/images/:id/url`;
5. abra a URL retornada dentro do tempo de expiração.

Se quiser testar pelo terminal:

```bash
curl -X POST http://IP_DA_EC2/api/images \
  -F "image=@/caminho/para/foto.png"
```

Depois:

```bash
curl http://IP_DA_EC2/api/images/1/url
```

## 9. Resultado esperado

```text
Usuário envia imagem
        ↓
Frontend encaminha para /api
        ↓
Backend recebe o arquivo
        ↓
Backend envia para o S3
        ↓
Backend salva a storage_key no RDS
        ↓
Backend retorna o cadastro do upload
```

Ao final desta etapa, você deve conseguir:

* fazer upload de uma imagem sem usar o disco da EC2 como armazenamento principal;
* encontrar o arquivo no bucket;
* salvar a chave do objeto no PostgreSQL do RDS;
* gerar uma URL temporária para acesso privado;
* preparar a base para etapas futuras, como thumbnails com Lambda ou distribuição com CloudFront.
