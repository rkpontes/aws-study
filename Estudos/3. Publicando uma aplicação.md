# Etapa 3 — Publicando uma aplicação na EC2

Vamos publicar uma aplicação simples com esta arquitetura:

```text
Internet
   ↓ porta 80
EC2
   ↓
Frontend (Nginx)
   ↓ /api
Backend (Node.js)
```

Usaremos Docker Compose. Somente o frontend ficará exposto à internet; o backend será acessível apenas pela rede interna do Docker.

---

## 1. Acesse a EC2

Assim como feito em [Etapa 2](./2.%20Seu%20primeiro%20servidor%20na%20AWS.md):

No seu computador:

```bash
chmod 400 aws-study-key.pem

ssh -i aws-study-key.pem ubuntu@IP_PUBLICO_DA_EC2
```

Exemplo:

```bash
ssh -i aws-study-key.pem ubuntu@54.123.45.67
```

Para instâncias Ubuntu, normalmente o usuário SSH é `ubuntu`. A chave privada precisa corresponder à chave pública configurada durante a criação da instância. ([Documentação AWS][1])

Confirme que o Docker está funcionando:

```bash
docker --version
docker compose version
```

---

## 2. Crie a estrutura do projeto

Na EC2:

```bash
mkdir -p ~/etapa-3-app/backend
mkdir -p ~/etapa-3-app/frontend
cd ~/etapa-3-app
```

A estrutura será:

```text
etapa-3-app/
├── backend/
│   ├── Dockerfile
│   ├── package.json
│   └── server.js
├── frontend/
│   ├── Dockerfile
│   ├── index.html
│   └── nginx.conf
└── compose.yaml
```

## 3. Crie o backend Node.js

Crie o `package.json`:

```bash
cat > backend/package.json <<'EOF'
{
  "name": "aws-etapa-3-backend",
  "version": "1.0.0",
  "private": true,
  "main": "server.js",
  "scripts": {
    "start": "node server.js"
  },
  "dependencies": {
    "express": "^5.1.0"
  }
}
EOF
```

Crie o servidor:

```bash
cat > backend/server.js <<'EOF'
const express = require("express");

const app = express();
const port = 3000;

app.get("/health", (request, response) => {
  response.json({
    status: "ok",
    service: "backend"
  });
});

app.get("/api/message", (request, response) => {
  response.json({
    message: "Backend executando na AWS EC2!",
    timestamp: new Date().toISOString()
  });
});

app.listen(port, "0.0.0.0", () => {
  console.log(`Backend executando na porta ${port}`);
});
EOF
```

Crie o Dockerfile:

```bash
cat > backend/Dockerfile <<'EOF'
FROM node:22-alpine

WORKDIR /app

COPY package*.json ./
RUN npm install --omit=dev

COPY server.js ./

EXPOSE 3000

CMD ["npm", "start"]
EOF
```

## 5. Configure o Nginx

O Nginx fará duas coisas:

1. Entregará o frontend.
2. Encaminhará requisições `/api` para o backend.

Crie o arquivo:

```bash
cat > frontend/nginx.conf <<'EOF'
server {
    listen 80;
    server_name _;

    root /usr/share/nginx/html;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location /api/ {
        proxy_pass http://backend:3000/api/;

        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
EOF
```

Crie o Dockerfile do frontend:

```bash
cat > frontend/Dockerfile <<'EOF'
FROM nginx:alpine

COPY index.html /usr/share/nginx/html/index.html
COPY nginx.conf /etc/nginx/conf.d/default.conf

EXPOSE 80
EOF
```

## 6. Crie o Docker Compose

Na raiz do projeto:

```bash
cat > compose.yaml <<'EOF'
services:
  backend:
    build:
      context: ./backend
    container_name: etapa-3-backend
    restart: unless-stopped
    expose:
      - "3000"
    healthcheck:
      test:
        - CMD-SHELL
        - wget -qO- http://127.0.0.1:3000/health >/dev/null 2>&1 || exit 1
      interval: 30s
      timeout: 5s
      retries: 3
      start_period: 10s
    networks:
      - app-network

  frontend:
    build:
      context: ./frontend
    container_name: etapa-3-frontend
    restart: unless-stopped
    depends_on:
      backend:
        condition: service_healthy
    ports:
      - "80:80"
    networks:
      - app-network

networks:
  app-network:
    driver: bridge
EOF
```

Observe a diferença:

```yaml
backend:
  expose:
    - "3000"
```

A porta `3000` fica disponível somente para outros containers da rede.

```yaml
frontend:
  ports:
    - "80:80"
```

A porta `80` da EC2 é encaminhada para a porta `80` do Nginx.

## 6.1. Ou faça o rsync

O mais prático é usar **`rsync` sobre SSH**. Ele envia apenas arquivos alterados, diferente do `scp`, que tende a copiar tudo novamente.

Da sua máquina local:

```bash
rsync -avz \
  -e "ssh -i ~/.ssh/aws-study-key.pem" \
  ./minha-aplicacao/ \
  ubuntu@IP_PUBLICO_DA_EC2:/home/ubuntu/minha-aplicacao/
```

Exemplo:

```bash
rsync -avz \
  -e "ssh -i ~/Downloads/aws-study-key.pem" \
  ~/Projetos/etapa-3-app/ \
  ubuntu@[IP_PUBLICO_DA_EC2]:/home/ubuntu/etapa-3-app/
```

```bash
rsync -avz \
  --exclude node_modules \
  --exclude .git \
  --exclude .next \
  --exclude dist \
  --exclude .env \
  -e "ssh -i ~/.ssh/aws-study-key.pem" \
  ~/Projetos/etapa-3-app/ \
  ubuntu@[IP_PUBLICO_DA_EC2]:/home/ubuntu/etapa-3-app/ \
&& ssh -i ~/.ssh/aws-study-key.pem ubuntu@[IP_PUBLICO_DA_EC2] \
  "cd /home/ubuntu/etapa-3-app && docker compose up -d --build"
```

A barra no final da origem é importante:

```bash
~/Projetos/etapa-3-app/
```

Ela significa: copie **o conteúdo da pasta**. Sem a barra, o `rsync` pode criar uma pasta adicional dentro do destino.

Execute:

```bash

```

```bash
docker compose ps
```



## 9. Abra a porta 80 no Security Group

No console da AWS:

```text
EC2
→ Instances
→ selecione a instância
→ Security
→ clique no Security Group
→ Edit inbound rules
→ Add rule
```

Adicione:

| Tipo | Protocolo | Porta | Origem      |
| ---- | --------- | ----: | ----------- |
| HTTP | TCP       |    80 | `0.0.0.0/0` |

Caso esteja usando IPv6, também poderá adicionar:

| Tipo | Protocolo | Porta | Origem |
| ---- | --------- | ----: | ------ |
| HTTP | TCP       |    80 | `::/0` |

O Security Group funciona como um firewall virtual da instância. Sem uma regra de entrada, o tráfego externo não chega à aplicação. ([Documentação AWS][2])

Para SSH, prefira restringir a porta 22 ao seu próprio IP:

| Tipo | Porta | Origem      |
| ---- | ----: | ----------- |
| SSH  |    22 | `SEU_IP/32` |
| HTTP |    80 | `0.0.0.0/0` |

Não é recomendado deixar SSH aberto em `0.0.0.0/0`.

## Resultado da etapa

Ao terminar, você terá:

```text
Internet
   ↓
Security Group — porta 80
   ↓
EC2
   ↓
Docker
   ├── Nginx/Frontend
   └── Backend Node.js
```

Você terá praticado:

* Publicação de backend e frontend.
* Docker Compose.
* Comunicação entre containers.
* Reverse proxy com Nginx.
* Mapeamento de portas.
* Security Groups.
* Acesso público por IPv4.
