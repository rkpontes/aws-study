# Projeto Único

Vamos desenvolver uma aplicação simples durante todo o estudo.

Exemplo:

> Sistema de Upload de Arquivos

Funcionalidades:

* Login
* Dashboard
* Upload de imagens
* Lista de arquivos
* Download
* Compartilhamento

Não importa muito qual sistema seja. O importante é que ele permita utilizar praticamente todos os serviços da AWS.

---

# Etapa 1 — Conhecendo a AWS

## Objetivo

Entender como a AWS é organizada.

### Estudar

* Regions
* Availability Zones
* IAM
* Console
* Billing
* Free Tier

### Prática

* Criar uma conta.
* Ativar MFA.
* Criar um usuário IAM (não usar a conta root).
* Configurar AWS CLI.

### Resultado

Você consegue acessar a AWS de forma segura.

# Como estudar cada etapa

Para cada uma, siga sempre a mesma rotina:

1. **Entenda o problema:** por que esse serviço existe?
2. **Estude a teoria:** conceitos, componentes e limitações.
3. **Implemente:** configure o serviço e integre à aplicação.
4. **Teste:** simule falhas, cargas ou cenários reais.
5. **Documente:** anote decisões, custos, vantagens e quando usar.

Assim, você não apenas aprende os serviços da AWS, mas também desenvolve a capacidade de escolher a ferramenta certa para resolver problemas reais, que é exatamente o que se espera de um desenvolvedor em ambiente de produção.
